# 정기차량 관리 자동화 프로그램 — 설계 문서

- 작성일: 2026-06-02
- 형태: Windows 데스크톱 GUI 프로그램 (PySide6) → PyInstaller 단일 exe
- 목적: 일반정기차량 목록에 제시기록대장 차량을 추가하고, 매도기록대장·오토넷출차리스트 차량을 제거한 뒤, 남은 중복을 사용자가 대화형으로 정리하여 최종 CSV를 생성한다. (2026-06-02에 수동으로 수행한 작업의 자동화)

## 1. 입력 파일

| 입력 | 개수 | 형식 | 비고 |
|------|:---:|------|------|
| 일반정기차량 | 1 | CSV | 처리의 기준(베이스) |
| 매도기록대장 | 1 | xlsx | 제거 대상 |
| 제시기록대장 | 1 | xlsx | 추가 대상 |
| 오토넷출차리스트 | 여러 개 | xlsx | 제거 대상 |

### 1.1 일반정기차량 (CSV) 양식 — 16열, 출력도 동일 양식
순서: `유효 / VIP / 차량번호 / 입주사아이디 / 입주사명 / 유효일시 / 만료일시 / 정기권명 / 월정액 / 결제금액 / 비고 / 부서명 / 이름 / 전화번호 / 메모 / 수정일시`
- 행1 = 헤더, 행2 = 설명행 `(O:추가 X:삭제)`, `(필수)`, `(선택)` 등, 행3~ = 데이터
- **차량번호 = 3번째 열**. 키로 사용한다.
- 헤더 2줄은 그대로 보존한다.
- **인코딩**: 입력 CSV 인코딩을 자동 감지(UTF-8 / UTF-8-SIG / CP949)하고, **출력은 입력과 동일한 인코딩·동일 헤더 2줄 구조**로 저장한다.

### 1.2 매도·제시 기록대장 (xlsx)
순서(2026.6 기준 9열): `일련번호 / 차종 / 등록번호 / 소유자 / 제시신고일 / 매도신고일 / 구분 / 매매업자(사업자) / 대장상태`
- 행1 = 헤더, 행2~ = 데이터
- 사용 컬럼: **등록번호**(키), **매도신고일**(공란 여부 판정), **매매업자(사업자)**(입주사명으로), **대장상태**
- 컬럼은 헤더명으로 찾는다(구버전 8열·`구분` 없음 호환). `구분` 컬럼은 사용하지 않는다.

### 1.3 오토넷출차리스트 (xlsx) — 파일마다 양식 상이
- 헤더행 위치·`차량번호` 열 위치가 파일마다 다르다.
- **상위 5행 내에서 `차량번호` 헤더 셀을 자동 탐색** → 그 아래 행부터 차량번호 추출(빈 행 skip).

## 2. 처리 로직

기호: A=일반정기차량 차량번호 집합, P=제시추가, M=매도, O=출차

1. 일반정기차량 CSV 로드 → A (행/정보 보존)
2. 제시대장에서 **제시추가(P)** 추출:
   - `대장상태 == '제시(상사)'` **또는** (`대장상태 == '미이전'` **그리고** `매도신고일`이 공란)
3. 매도대장 전체 등록번호 → **M**
4. 출차리스트들의 차량번호 합집합 → **O**
5. **결과 = (A ∪ P) − M − O**
   - A의 행 중 차량번호가 (M ∪ O)에 속하면 **모든 해당 행 삭제**(중복 행 포함)
   - P 중 A에 없고 (M ∪ O)에도 없는 차량번호 → **새 행 추가**(아래 값 채움)
6. 결과에서 **차량번호 중복(2행 이상) 탐지** → 3단계 중복 해결로

### 2.1 추가 행 값
| 컬럼 | 값 |
|------|----|
| 유효 | `O` |
| VIP | `X` |
| 차량번호 | 해당 차량번호 |
| 입주사명 | 제시대장의 `매매업자(사업자)` 값 |
| 유효일시 | 기준일 `YYYY-MM-DD 00:00:00` |
| 만료일시 | `2999-12-31 00:00:00` |
| 그 외(입주사아이디·이름·전화 등) | 공란 |

기준일: 프로그램 실행 시 입력(기본값 = 오늘 날짜).

## 3. 화면 흐름 (PySide6, 3단계)

### [1단계] 파일 선택
- 일반정기차량 CSV / 매도대장 / 제시대장 각 1개 선택 버튼 + 선택된 파일명 표시
- 오토넷출차리스트 `+추가`(다중), 추가된 목록 표시 + 개별 `x` 제거
- 기준일 입력(QDateEdit, 기본=오늘)
- `처리 시작` 버튼 (필수 입력 누락 시 비활성/경고)

### [2단계] 중복 해결 (남은 중복이 있을 때만)
- 상단: 처리 요약 `제시추가 +N / 매도삭제 −N / 출차삭제 −N / 남은 중복 N그룹`
- 중복 그룹마다 카드/섹션:
  - 각 행을 **라디오(남길 행 1개 선택)** 로 나열, 입주사명은 **편집 가능 필드**
  - **`둘 다(모두) 삭제`** 체크박스
  - 입주사명이 모두 동일한 순수중복은 **첫 행을 기본 선택**으로 추천
- `최종 CSV 생성` 버튼

### [3단계] 저장
- 저장 위치 선택(QFileDialog) → 최종 CSV 작성 → 완료 요약(최종 N대) 표시

## 4. 코드 구조
- `parser.py` — 파일별 차량/정보 추출 (CSV·대장·출차, 헤더 자동 인식)
- `engine.py` — 집합 연산, 추가 행 생성, 중복 탐지, 중복 해결 적용, CSV 출력
- `gui/` — PySide6 화면(1·2·3단계), `main.py` 진입점
- `engine.py`/`parser.py`는 GUI와 분리(독립 테스트 가능)
